# Проект "Приключенец - Моят Пътеводител" – Контейнеризация с Docker Compose

Това ръководство описва процеса по контейнеризация на уеб приложението "Приключенец". Приложението е разработено на PHP и служи за споделяне и разглеждане на интересни дестинации, маршрути и забележителности за пътуване. Системата използва MySQL база данни за съхранение и управление на споделените локации. Всички компоненти са конфигурирани да работят съвместно в изолирана Docker среда чрез **Docker Compose**.

---

## Проектна структура

Файловете в проекта са разпределени по следния начин:
* `src/` — съдържа основните PHP скриптове (`index.php` и `db.php`).
* `styles/` — съдържа CSS стиловете за визуализация на потребителския интерфейс (`style.css`).
* `kursova_2.sql` — базов SQL скрипт за инициализация на базата данни.
* `db_init/` — директория за автоматично импортиране на SQL базата данни при първоначално стартиране.
* `Dockerfile` — дефинира стъпките за компилиране на персонализирания PHP-Apache уеб сървър.
* `compose.yml` — управлява конфигурацията, обемите и свързаността между контейнерите.

---

## Архитектура на системата

Приложението е разделено на две основни изолирани услуги:

### 1. Уеб сървър (`web`)
* **Основа:** Официален образ `php:8.2-apache`.
* **Допълнителна конфигурация (Dockerfile):** Инсталиран е модулът `pdo_mysql`, който позволява на PHP да комуникира с MySQL.
* **Достъп:** Настроен е да приема заявки на порт `8080` (мапнат към порт `80` в контейнера).
* **Публикуван образ:** Готовият Docker образ е качен в Docker Hub: [brato4kata/kursova-web](https://hub.docker.com/r/brato4kata/kursova-web)

### 2. Сървър за база данни (`db`)
* **Основа:** `mysql:8.0`.
* **Съхранение на данните:** Използва се именуван обем (`mysql_data`), за да се предотвратят загуби на информация при спиране или премахване на контейнера.
* **Инициализация:** Чрез монтиране на `db_init/` към `/docker-entrypoint-initdb.d` се осигурява автоматично стартиране на SQL скрипта при първично вдигане на базата.

---

## Връзка и комуникация между услугите

Двата контейнера споделят обща виртуална мрежа, автоматично създадена от Docker Compose.

В конфигурационния файл `src/db.php` връзката с базата данни се осъществява динамично чрез следните параметри:
```php
$host = "db";
$dbname = "kursova_2";
$user = "root";
$password = "1234";
```

## Начин на стартиране

За да сглобите и стартирате проекта във вашата локална среда през WSL (Windows Subsystem for Linux):

### Вариант А: Автоматична инициализация на базата данни (Препоръчително)
1. Копирайте SQL файла в директорията `db_init/`:
   ```bash
   cp kursova_2.sql db_init/
   ```
2. Стартирайте услугите и изчистете старите обеми данни, за да принудите MySQL да се инициализира отново:
   ```bash
   docker compose down -v
   docker compose up --build -d
   ```
3. Достъпете приложението през браузъра си на адрес:
   ```text
   http://localhost:8080/src/index.php
   ```

### Вариант Б: Ръчно импортиране на базата данни
Ако стартирате проекта без копиране на файла в `db_init/`:
1. Стартирайте услугите:
   ```bash
   docker compose up --build -d
   ```
2. Създайте ръчно базата данни в контейнера (ако не е създадена):
   ```bash
   docker exec -it kursova-db-1 mysql -u root -p1234 -e "CREATE DATABASE IF NOT EXISTS kursova_2;"
   ```
3. Импортирайте SQL файла ръчно:
   ```bash
   docker exec -i kursova-db-1 mysql -u root -p1234 kursova_2 < kursova_2.sql
   ```
   *(Забележка: Ако контейнерът се казва по друг начин (например `kursova_db_1`), използвайте правилното име от `docker ps`).*
